# Roadmap: Punal Manalan Portfolio

## Overview

This roadmap turns the empty portfolio scaffold into a finished personal site in three passes: establish accurate content and structure, refine the visual and responsive experience, then close with metadata and final polish.

## Phases

**Phase Numbering:**

- Integer phases (1, 2, 3): Planned milestone work
- Decimal phases (2.1, 2.2): Urgent insertions (marked with INSERTED)

Decimal phases appear between their surrounding integers in numeric order.

- [ ] **Phase 1: Portfolio Content Foundation** - Replace starter content with the full single-page portfolio structure and authored resume content.
- [ ] **Phase 2: Visual Identity and Responsive Experience** - Shape the portfolio into a dark, polished, mobile-friendly experience with subtle interaction cues.
- [ ] **Phase 3: Metadata and Final Polish** - Add metadata and finish the page for a clean first release.

## Phase Details

### Phase 1: Portfolio Content Foundation

**Goal**: Replace the default starter page with a semantic single-page portfolio containing the core profile, experience, and skills narrative.
**Depends on**: Nothing (first phase)
**Requirements**: [PROF-01, PROF-02, EXP-01, EXP-02, SKIL-01, UI-01, DELV-01, DELV-02]
**Success Criteria** (what must be TRUE):

1. Visitor can see a hero section with Punal Manalan's name, headline, and concise intro.
2. Visitor can read about, experience, and technical arsenal sections with real authored content.
3. The default Next.js starter page is fully removed.
4. The page uses semantic HTML structure inside the App Router page.
   **Plans**: 1 plan

Plans:

- [ ] 01-01: Deliver the authored semantic content foundation for the portfolio landing page

### Phase 2: Visual Identity and Responsive Experience

**Goal**: Turn the content foundation into a distinctive dark portfolio with responsive layout behavior and restrained interactivity.
**Depends on**: Phase 1
**Requirements**: [SKIL-02, UI-02, UI-03, UI-04, PRES-01]
**Success Criteria** (what must be TRUE):

1. Visitor sees a dark, modern visual system with strong hierarchy and readable typography.
2. The portfolio adapts cleanly across mobile, tablet, and desktop breakpoints.
3. Interactive elements provide clear hover and focus feedback.
4. The hero offers a clear path into the rest of the page.
   **Plans**: 2 plans

Plans:

- [ ] 02-01: Establish dark visual system, section surfaces, and typography rhythm
- [ ] 02-02: Tune responsive layout and add CTA plus interaction states

### Phase 3: Metadata and Final Polish

**Goal**: Add portfolio-specific metadata and finish the page with launch-readiness checks.
**Depends on**: Phase 2
**Requirements**: [PRES-02]
**Success Criteria** (what must be TRUE):

1. Browser title and description identify the page as Punal Manalan's portfolio.
2. The page feels release-ready with no leftover starter artifacts.
3. Final quality checks confirm the page remains coherent after polish.
   **Plans**: 1 plan

Plans:

- [ ] 03-01: Add metadata and perform final portfolio polish pass
