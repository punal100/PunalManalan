# Requirements: Punal Manalan Portfolio

**Defined:** 2026-04-14
**Core Value:** Present Punal Manalan as a credible software developer and R&D-focused builder through a polished, fast, mobile-friendly portfolio page.

## v1 Requirements

Requirements for the initial portfolio release. Each maps to roadmap phases.

### Profile

- [ ] **PROF-01**: Visitor can immediately see Punal Manalan's name and professional headline in the hero section.
- [ ] **PROF-02**: Visitor can read an about section summarizing R&D background and areas of interest.

### Experience

- [ ] **EXP-01**: Visitor can view the OpEzee R&D Developer role with the March 2026 transition note.
- [ ] **EXP-02**: Visitor can understand the role focus on bridging research concepts and technical implementation.

### Skills

- [ ] **SKIL-01**: Visitor can browse technical skills grouped by languages, engines, mobile development, AI and machine learning, and tools.
- [ ] **SKIL-02**: Visitor sees subtle hover or focus feedback on skill or category treatments.

### Interface

- [ ] **UI-01**: Visitor can move through a single-page layout with hero, about, experience, and technical arsenal sections.
- [ ] **UI-02**: Visitor sees a dark, modern interface with legible typography and clear visual hierarchy.
- [ ] **UI-03**: Visitor can comfortably use the portfolio on mobile, tablet, and desktop layouts.
- [ ] **UI-04**: Visitor receives clear hover and focus states on interactive elements.

### Delivery

- [ ] **DELV-01**: Visitor loads the portfolio from a Next.js App Router page implemented with semantic HTML and Tailwind CSS.
- [ ] **DELV-02**: Visitor no longer sees the default Next.js starter content.

### Presence

- [ ] **PRES-01**: Visitor can jump from the hero section to deeper portfolio sections through a clear call to action.
- [ ] **PRES-02**: Visitor receives descriptive page metadata suitable for a personal portfolio.

## v2 Requirements

### Portfolio Growth

- **GROW-01**: Visitor can browse featured project case studies.
- **GROW-02**: Visitor can download a PDF resume.
- **GROW-03**: Visitor can contact Punal Manalan through a dedicated contact flow.

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature                          | Reason                                                   |
| -------------------------------- | -------------------------------------------------------- |
| Blog or CMS                      | Not required for the first portfolio release             |
| Authentication or admin area     | No user accounts or content management are needed        |
| Database-backed content editing  | Static authored content is sufficient for v1             |
| Multi-page application structure | The current content fits a single, stronger landing page |
