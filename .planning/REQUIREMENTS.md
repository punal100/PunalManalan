# Requirements: Punal Manalan Portfolio

**Defined:** 2026-04-14
**Core Value:** Present Punal Manalan as a credible software developer and R&D-focused builder through a polished, fast, mobile-friendly portfolio page.

Requirements for the initial portfolio release. Each maps to roadmap phases.

### Profile

- [x] **PROF-01**: Visitor can immediately see Punal Manalan's name and professional headline in the hero section.
- [x] **PROF-02**: Visitor can read an about section summarizing R&D background and areas of interest.
- [x] **PROF-03**: Visitor can access public LinkedIn and GitHub links from the portfolio page.

### Experience

- [x] **EXP-01**: Visitor can view the OpEzee R&D Developer role with the March 2026 transition note.
- [x] **EXP-02**: Visitor can understand the role focus on bridging research concepts and technical implementation.
- [x] **EXP-03**: Visitor can scan additional experience across Pixis AI, Wrexa Technologies, and freelance Unreal Engine work.

### Featured Work

- [x] **WORK-01**: Visitor can read a featured summary of the Wrexa xMall 3D Meet / virtual meeting platform project.

### Open Source

- [x] **OSS-01**: Visitor can scan selected open-source projects and achievements.
- [x] **OSS-02**: Visitor can browse grouped additional Unreal Engine public repositories and tooling.
- [x] **OSS-03**: Visitor can see notable external contribution work across public repositories.
- [x] **OSS-04**: Visitor can see legacy API-dependent projects with enough context to understand their historical role.

### Skills

- [x] **SKIL-01**: Visitor can browse technical skills grouped into clear domains drawn from the resume, including systems languages, Unreal and 3D, networking/backend, AI/tooling, and platforms.

### Interface

- [x] **UI-01**: Visitor can move through a single-page layout with hero, about, experience, featured work, open-source, and technical arsenal sections.
- [x] **UI-02**: Visitor sees a dark, modern interface with legible typography and clear visual hierarchy.

- [x] **UI-04**: Visitor receives clear hover and focus states on interactive elements.

### Presence

- [x] **PRES-01**: Visitor can jump from the hero section to deeper portfolio sections through a clear call to action.
- [ ] **PRES-02**: Visitor receives descriptive page metadata suitable for a personal portfolio.

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature                          | Reason                                                   |
| -------------------------------- | -------------------------------------------------------- |
| Blog or CMS                      | Not required for the first portfolio release             |
| Authentication or admin area     | No user accounts or content management are needed        |
| Database-backed content editing  | Static authored content is sufficient for v1             |
| Multi-page application structure | The current content fits a single, stronger landing page |
